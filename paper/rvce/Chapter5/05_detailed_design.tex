\chapter{Detailed Design}

The Detailed Design phase provides an in-depth description of the internal structure, modules, and operational workflow of the proposed system. It focuses on the decomposition of the system into smaller functional components and explains how each module contributes to achieving the overall objectives of the project. This chapter presents the structure chart, module descriptions, and the detailed functionality of individual components involved in the implementation. The chapter concludes with a summary of the design specifications considered during development.

%----------------------------------------------------------

\section{Structure Chart}

A structure chart represents the hierarchical organization of modules within the proposed system. It illustrates the interaction and communication between different functional components and provides a clear understanding of the system workflow.

The structure chart may include:
\begin{itemize}
    \item Main system modules
    \item Sub-modules and their relationships
    \item Data flow between modules
    \item Control hierarchy
\end{itemize}

\section{Module Description}

This section provides a detailed explanation of the individual modules present in the system. Each module should be described with its functionality, inputs, outputs, and role within the overall architecture.

The module descriptions may include:
\begin{itemize}
    \item Purpose of the module
    \item Functional workflow
    \item Input and output specifications
    \item Algorithms or techniques used
    \item Interaction with other modules
\end{itemize}

%----------------------------------------------------------

\subsection{Mapper}

The Mapper module is responsible for processing input data and transforming it into intermediate key-value pairs suitable for further computation. This module performs data partitioning, preprocessing, and mapping operations required for distributed processing or analytical computation.

The Mapper module may include:
\begin{itemize}
    \item Data parsing and preprocessing
    \item Key-value pair generation
    \item Data filtering and transformation
    \item Intermediate result generation
\end{itemize}

\subsection{Reducer}

The Reducer module processes the intermediate outputs generated by the Mapper module and performs aggregation, summarization, or analytical operations to produce the final result.

The Reducer module may include:
\begin{itemize}
    \item Data aggregation
    \item Result computation
    \item Filtering and optimization
    \item Final output generation
\end{itemize}

\subsection{Selection of Centroids}

This module explains the process of selecting centroids during clustering or analytical operations. The centroid selection process plays a crucial role in determining the efficiency and accuracy of clustering algorithms such as K-Means.

The discussion may include:
\begin{itemize}
    \item Initial centroid selection methods
    \item Distance calculation techniques
    \item Cluster assignment process
    \item Iterative centroid updates
    \item Convergence conditions
